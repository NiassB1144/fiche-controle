# 🚀 GUIDE INTÉGRATION - Offline v4 (Solution finale)

## ⚡ TL;DR (Faire en 2 minutes)

### 1. Dans votre `base.html` ou `layout.html`

Remplacer TOUS les anciens scripts offline par celui-ci:

```html
{% load static %}

<!-- À la fin du <body> -->
<script src="{% static 'js/offline-v4.js' %}"></script>
```

**C'est tout!** ✓

## 🎯 Maintenant ça fonctionne

### ✅ Voir les détails offline

```
1. Consulter une fiche en ligne (vous êtes automatiquement connecté)
2. Les données sont cachées en IndexedDB
3. Aller offline (DevTools → Network → Offline)
4. Cliquer sur la même fiche
5. ✓ Les détails s'affichent!
```

### ✅ Modifier une fiche offline

```
1. Être sur une page de détail/modification
2. Modifier les champs
3. Aller offline
4. Cliquer "Sauvegarder"
5. ✓ Message "Fiche modifiée en offline"
6. Aller online
7. ✓ Auto-sync après quelques secondes
```

### ✅ Créer une fiche offline

```
1. Aller sur le formulaire de création
2. Aller offline
3. Remplir le formulaire
4. Cliquer "Créer"
5. ✓ Message "Fiche créée en offline"
6. Aller online
7. ✓ Auto-sync
```

### ✅ Supprimer une fiche

**Nota:** Pour les DELETE, il faut être online (limitation browser)
Ou utiliser la modification offline + supprimer online.

## 📊 Fonctionnement

### Processus complet

```
PAGE HTML
   ↓
Script offline-v4.js charge
   ↓
Détecte les fiches sur la page
   ↓
Cache en IndexedDB
   ↓ Vous allez offline
Service Worker intercepte requests
   ↓
Cherche en cache (IndexedDB)
   ↓ Si modification
Formulaire est intercepté
   ↓
Sauvegarde en IndexedDB
   ↓
Ajoute à sync_queue
   ↓ Vous allez online
Auto-sync se déclenche
   ↓
Envoie à /api/sync/
   ↓
Serveur traite
   ↓
Item marqué "synced"
   ↓
✓ Fini!
```

## ⚠️ Limitations

### ✅ Fonctionne
- Voir une fiche consultée avant
- Modifier une fiche consultée avant
- Créer des fiches offline
- Auto-sync modification/création

### ❌ Ne fonctionne pas
- Voir une fiche JAMAIS consultée avant (normal - pas en cache)
- Supprimer offline (limitation browser DELETE)

### Solution workaround
```
1. Pré-charger les fiches importantes:
   window.OfflineSystem.getAllFiches()  // Force le cache

2. Ou mettre dans un <script> au chargement:
   <script>
     // Pré-cache toutes les fiches fréquemment utilisées
     fetch('/api/fiches/').then(r => r.json());
   </script>
```

## 🐛 Troubleshooting

### "Fiche non trouvée" offline

**Problème:** Essayez de voir une fiche jamais consultée

**Solution:**
```
1. Aller online
2. Cliquer sur la fiche pour la consulter
3. Puis aller offline
4. Réessayer
```

### Modifications ne synchronisent pas

**Vérifier:**
```
1. Devtools → Console → Erreurs? (F12)
2. Vérifier online: window.OfflineSystem.isOnline()
3. Vérifier sync queue: 
   const items = await window.OfflineSystem.getAllFiches()
4. Forcer sync: window.OfflineSystem.sync()
```

### Service Worker pas enregistré

**Solution:**
```
1. Devtools → Application → Service Workers
2. Cliquer "Unregister"
3. Recharger (Ctrl+Shift+R ou Ctrl+F5)
4. Doit se re-enregistrer
```

### Offline panel ne s'affiche pas

**Solution:**
```
1. Aller offline (Devtools → Network → Offline)
2. Recharger
3. Panel orange doit apparaître en haut
```

## 💡 Tips avancés

### Pré-cacher les fiches au démarrage

```html
<!-- Ajouter dans votre template -->
<script>
  if (window.OfflineSystem) {
    // Au chargement, récupérer et cacher toutes les fiches
    fetch('/api/fiches/')
      .then(r => r.json())
      .then(data => console.log(`✓ ${data.length} fiches pré-chargées`))
      .catch(e => console.warn('Pré-cache échoué:', e));
  }
</script>
```

### Déboguer les données IndexedDB

```javascript
// Voir TOUTES les fiches
const fiches = await window.OfflineSystem.getAllFiches();
console.table(fiches);

// Voir les items en attente de sync
const db = (await indexedDB.open('FicheControleDB')).result;
const tx = db.transaction(['sync_queue'], 'readonly');
const req = tx.objectStore('sync_queue').getAll();
req.onsuccess = () => console.table(req.result);
```

## 📞 Support

### Script ne charge pas?

```
1. Vérifier chemin correct: /static/js/offline-v4.js
2. Vérifier 200 OK en Devtools → Network
3. Vérifier pas d'erreur console
4. Vérifier base.html load le script
```

### Cache ne fonctionne pas?

```
1. Vérifier IndexedDB accessible
2. Vérifier Service Worker enregistré
3. Vérifier pas d'erreurs console
4. Essayer: localStorage.clear() puis recharger
```

### Sync échoue?

```
1. Vérifier online: window.OfflineSystem.isOnline()
2. Vérifier /api/sync/ retourne 200
3. Vérifier CSRF token valide
4. Vérifier Django logs pour erreurs
```
