THE PROBLEM SOLVED
================================================================================

BEFORE (❌ BROKEN):
  ✅ Create fiche offline → Works
  ❌ View fiche offline → Django 404 Error
  ❌ Modify fiche offline → Django 404 Error
  ❌ Delete fiche offline → Django 404 Error

REASON:
  When offline, fiches were created with local_id but URLs pointed to
  Django routes that required server connectivity. Django couldn't be
  reached, so all operations failed with 404 errors.

AFTER (✅ FIXED):
  ✅ Create fiche offline → Works (IndexedDB)
  ✅ View fiche offline → Works (load from cache)
  ✅ Modify fiche offline → Works (update IndexedDB)
  ✅ Delete fiche offline → Works (delete from IndexedDB)
  ✅ Automatic sync when online → Works (sync to Django)

================================================================================
HOW IT WORKS
================================================================================

OFFLINE MODE:
  1. User creates fiche
  2. JavaScript stores in IndexedDB (not Django!)
  3. Shows in list with "🗄️ Hors-ligne" badge
  4. User clicks "Voir" → offline-fiche.html page loads from cache
  5. Page loads data directly from IndexedDB (zero Django calls!)
  6. User can modify or delete within IndexedDB
  7. All changes are 100% offline

ONLINE MODE:
  1. Service Worker detects connectivity
  2. Background sync starts automatically
  3. Fiches are sent to Django API
  4. Django creates fiches and assigns IDs
  5. IDs are stored locally (server_pk)
  6. Marked as synced=true
  7. User sees "✅ Synchronisée" status

TECHNICAL STACK
================================================================================

Frontend:
  • HTML5 + CSS3 (responsive design)
  • Vanilla JavaScript (no dependencies except offline-crud.js)
  • Service Worker API (for caching & sync)
  • IndexedDB API (for local storage)

Backend:
  • Django 3.x+
  • Django REST Framework (API)
  • SQLite / PostgreSQL (database)

Architecture:
  • Offline-first (local first, sync later)
  • Cache-first strategy (static assets)
  • Network-first strategy (API calls)
  • Background sync (automatic when online)
