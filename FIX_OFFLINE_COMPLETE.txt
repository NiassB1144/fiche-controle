⚙️ COMMENT CA MARCHE

Flux création offline:
──────────────────
1. User crée fiche offline
2. offline-v4.js intercepte formulaire
3. Génère local_id = Date.now()
4. Sauvegarde en IndexedDB.fiches
5. Ajoute à sync_queue avec action="create"
6. Affiche "Fiche créée en offline" ✓

Flux accès fiche locale:
──────────────────────
1. User clique sur fiche (href="/fiche/local/123456789")
2. Navigateur va à /fiche/local/123456789/
3. Django → view_local_fiche()
4. Charge template view_local_fiche.html
5. JavaScript côté client:
   - Appelle window.OfflineSystem.getFiche('123456789')
   - IndexedDB retourne les données
   - Affiche en HTML ✓

Flux modification offline:
───────────────────────
1. User clique "Modifier"
2. Modal apparaît avec formulaire
3. User change champ
4. Clique "Sauvegarder"
5. Données sauvegardées en IndexedDB
6. Ajouté à sync_queue avec action="update"
7. Affiche "Fiche modifiée!" ✓

Flux sync:
─────────
1. User revient online
2. offline-v4.js détecte "online"
3. Appelle syncPending()
4. Pour chaque item dans sync_queue:
   POST /api/sync/
   {
     "action": "create|update|delete",
     "data": {...}
   }
5. Django process, crée/modifie en BD
6. Marque item comme "synced"
7. ✓ Fiche maintenant a un id serveur
